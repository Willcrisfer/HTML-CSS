## Objectivos:

1 - Reproduzir estruturas de html para os cartões no figma.
2 - Separação de responsabilidades. Cada cartão no seu próprio ficheiro.

## Pontos a ter em conta:

1 - Semântica.
O HTML deve usar elementos semânticos apropriados para representar o conteúdo. Isso ajuda não apenas na compreensão do código, mas também na acessibilidade e no SEO.

2 - Utilização da hierarquia certa de elementos:
Utiliza hierarquia de títulos (<h1>, <h2>, etc.) de forma apropriada para estruturar o conteúdo e fornecer uma leitura clara da página.

3 - Eficiência.
Mantêm o teu html limpo. Define estruturas com clareza e comenta o teu código. Isto ajuda a identificar o propósito de cada elemento.

4 - Acessibilidade.
Inclua atributos alt nos elementos multimédia para descrever o conteúdo. Fornece alternativas acessíveis para elementos multimédia, como vídeos e áudios.

Lembra-te de que um código HTML bem feito não é apenas benéfico para a manutenção do teu site, mas também para a experiência do utilizador e a acessibilidade. Adotar boas práticas desde o início pode poupar-te tempo e evitar problemas no futuro.
